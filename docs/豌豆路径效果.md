# 豌豆路径后续效果

## 目标

底排豌豆的前半段路径保持统一：先向左移动到草坪最左侧，再向上移动到 row 0。到达 row 0 最左侧后的行为由关卡配置决定，用于把不同物理实验隔离成不同关卡。

## 关卡配置

关卡 RON 使用 `pea_path_arrival_effect` 选择效果：

```ron
(
    id: "level_01",
    display_name: "前院防线",
    always_shoot: true,
    pea_path_arrival_effect: Straight,
    // ...
)
```

当前效果：

- `Straight`: 保留当前行为；到达 row 0 后继续向右飞行。
- `RowThreePhysicsLine`: 到达 row 0 后销毁原路径豌豆，在 row 3 高度从最左到最右生成 20 个物理豌豆。

默认关卡 `assets/levels/level_01.ron` 使用 `Straight`。物理实验关卡为 `assets/levels/level_row_three_physics_line.ron`，该关卡保持默认关卡的 12 波结构，但僵尸数量更多、间隔更短，并开启 `gatling_pea_upgrade_only`，要求机枪射手只能替换多发射手。

运行物理实验关卡：

```powershell
cargo run -- --level assets/levels/level_row_three_physics_line.ron
```

## 代码结构

关卡配置读入 `LevelDefinition.pea_path_arrival_effect`：

```rust
pub struct LevelDefinition {
    pub always_shoot: bool,
    pub pea_path_arrival_effect: PeaPathArrivalEffect,
    // ...
}
```

植物发射时只负责把关卡选项写入路线，不处理具体效果：

```rust
ProjectileRoute::LeftEdgePath {
    turn_x: layout.origin.x,
    target_y: layout.path_y() + muzzle_offset.y,
    after_arrival: definition.pea_path_arrival_effect,
}
```

弹丸系统推进公共路径；只有到达 row 0 最左侧时才分发后续效果：

```rust
match route.after_arrival {
    PeaPathArrivalEffect::Straight => {
        route.phase = LeftEdgePathPhase::MoveRight;
    }
    PeaPathArrivalEffect::RowThreePhysicsLine => {
        spawn_row_three_physics_line(...);
        commands.entity(entity).despawn();
    }
}
```

新增效果时优先新增 `PeaPathArrivalEffect` 枚举值和独立的 spawn/advance 函数，不要把效果细节写进植物发射逻辑。这样各关卡只通过 RON 选择效果，效果之间不会互相污染。

## 火豌豆处理

`RowThreePhysicsLine` 读取触发弹丸当前实体上的 `ProjectileKind` 和 `Projectile.damage`，再生成物理运动的同种弹丸。因此：

- 普通豌豆触发后生成物理普通豌豆。
- 寒冰豌豆触发后生成物理寒冰豌豆。
- 已穿过火炬的火豌豆触发后生成物理火豌豆，并保留 40 伤害、直接命中解冰和小范围溅射。

这里的“物理豌豆”指 `ProjectileMotion::Physics`，不强制使用 `ProjectileKind::PhysicsPea`。
